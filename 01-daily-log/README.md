# Daily log (`01-daily-log/`)

Lightweight **learning journal** for this repo: one Markdown file per calendar day so progress stays visible (especially alongside **`03-python-foundations/`**).

---

## What goes here

| Item | Meaning |
|------|---------|
| **`TEMPLATE.md`** | Copy into a new dated file when you sit down to study or build. |
| **`YYYY-MM-DD.md`** | One file per day (e.g. `2026-05-11.md`) — bullets only; keep friction low. |

This folder pairs with the habit described in **[`03-python-foundations/CURRICULUM-A-Z.md`](../03-python-foundations/CURRICULUM-A-Z.md)** (*practice loop*: read a topic → write a short snippet → log one bullet).

---

## Where it fits in the repo

| Resource | Role |
|----------|------|
| **[`03-python-foundations/README.md`](../03-python-foundations/README.md)** | **Tour** of exercises, LeetCode drills, OOP track, runnable script folders. |
| **[`03-python-foundations/exercises/README.md`](../03-python-foundations/exercises/README.md)** | Guided **01→22** Jupyter path — ideal notebook numbers to cite in your log. |
| **[`03-python-foundations/leetcode_practice/README.md`](../03-python-foundations/leetcode_practice/README.md)** | **`unittest`** pattern drills — note `easy`/`medium`/`hard` functions you cleared. |
| **[`START_HERE.md`](../START_HERE.md)** (repo root) | Orientation when switching tracks (e.g. toward **`04-api-engineering/`**). |

---

## How to start a day

1. Copy **`TEMPLATE.md`** → **`YYYY-MM-DD.md`** (today’s date).
2. Under **Today**, note what you’ll touch (e.g. “finish exercises `07-decorators-generators.ipynb`”).
3. After coding, fill **Shipped / built** with concrete artifacts (files changed, tests passed).
4. Use **Blockers** for errors or concepts to revisit; **Tomorrow** for the next smallest step.

Keep entries short — the goal is **continuity**, not perfection.
